Bit Projects

When bit is invoked, it searches for a start.bit file in the current directory. If one is not found, it searches up the parent directory chain for one. Bit then loads the nearest start.bit file and commences processing its build instructions. This means you can typically invoke bit from anywhere in a project source tree. A start.bit file may be complete in itself, or it may in turn load other bit files.

Use cases for Bit

There are two use cases of use for Bit:

  • Stand-alone Use
  • Configured Use

Stand-Alone Use

A stand-alone start.bit file may be used to express simple build targets that do not requiring any prior configuration step. The start.bit file contains targets that specify the recipes to build those targets. The full capabilities of bit are available and you can compile sources, build libraries and executables or run scripts.

Configured Use

Some products/projects require a configuration phase prior to building with Bit. Bit configuration of a project discovers the available tools and components and customizes the project to best run on the local system. For configured products, Bit is first invoked with a configure target. Bit then reads a main.bit file which describes the project, before it discovers the local tools and components. Finally, it generates a start.bit and platform bit file that describes the project build and system. Configured projects support cross-compilation and project file generation for use in IDEs.

To learn more, read about Compiling Sources.
